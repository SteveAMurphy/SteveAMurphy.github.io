Arduino

One of my electives for the first semester of my masters was Physical Computing.

I have always had an interest in automation and computer aided manufacture so the description of the module appealed to me as it involved the use of sensors and actuators, inputs and outputs and an element of coding.

Coming from a background of manufacture and more recently the area of product design, I think this module and the knowledge I have gained from it will stand to me in the future. More and more I am asked to help students develop their representative models into something a bit more interactive. I think the Arduino platform will be perfect for this.

It would be a lie if I said that I did not find the module tough. There was a very steep learning curve and most of the fundamentals of arduino coding were new to me. Coding of any kind is a new experience. But I am very happy with my output all things considered.

Physical Computing Major Project

For the final major project of my module on Physical Computing I designed and built an interactive game for kids aged 8 and upwards.

The intention behind the game was to use the skills leared in the module to design and build a system based on Arduino to address a topic of my choice. The project was a group project so our team decided on the topic of Games. From that general topic, I decided to explore using Arduino to develop an educational game. Math-Duino was the result.

My research led me to the sub-topic of mathematics. This is an area of primary education which lends itself well to the Arduino platform for developing kids mathematics skills. The players can choose between a Trigonometry game where they need to guess the angle of a needle and an Multiplication game to help them to revise their multiplication tables.

Not only does the system encourage kids to develop their mathematics skills, Math-Dunio also takes kids away from screens and touch based screen games and helps develop fine motor skills and spacial skills.

I have shared some of the steps of the development process below. As usual, there is some reflection on each step.

Early Ideation

Getting some ideas out of my head for discussion with the group.

After some brainstorming of ideas, I took some sketches I had generated created an Illustrator representation of them so I can discuss with the tutor and my group members. Although it takes some time, I feel that any genuine opportunity to use the software is worth it as it also helps develop my Illustrator knowledge.

Arduino Block Model

Figuring out the user flow through the Math-Dunio Game.

Using Mural software, I mapped out the different game-states of the user flow. This was a critial step in the development of the code for the game. Each game-state served as a stage-gate of block in the code. As the user worked their way through the game, different game-states were activated in the code to allow the Arduino to perform different sets of tasks for each.

Not only did this step in the process introduce me to new digital tools like Mural, it also forced me to walk through the final game design and critically think through each step.

Wiring

Paper and pen can't electrocute me!

To get my head around the wiring of the game, I decided to draw out a schematic of the wiring and system.

I think this is always a worth-while exercise as it enables me to communicate the plans with others. This will be beneficial in future projects with fellow students or if I ever need to work with electricians or engineers.

Arduino Wiring

Using Tinkercad to figure out the wiring of the system

As part of the process of developing the Math-Duino code, a wiring and component diagram was generated in Tinkercad. This system helped to figure out the connection before going onto the physical build. With the addition of code, the entire system could be tested virtually. A great way to iron out any issues before commiting to wiring and reducing the risk of shorting any of the limited components needed for the build.

Prototyping

Get it working, then get it looking pretty.

The image above is not usually shared. It's a terrible image, but it does represent the prototyping stage pretty well. At this stage it's all about testing the system. There are many rapid changes to the set-up and the code. The configuration of the system is only just now coming together so there is no point being too structured and organised. Get it working and go from there.

I learned alot from this stage. I began by using the Arduino srtarter kit but I found that this Seeduino kit had quick connect wiring so it was much faster to re-configure.

Math-Duino Final Model

An interactive Mathematics Game

The final model aimed to make the Math-duino game presentable for grading but also if the system needed to be developed further, the model would prove a useful way of getting user testing.

All the Arduino components were attached to a mounting board and then the face-plate was attached. The face-plate was made from black 3mm acrylic which was laser engraved and cut to accomodate the interaction points and the displays.

As with any prototype, there is room for improvement. Given the opportiunity I plan to re-configure this system so that a user can pick it up and play it like a handheld game. As the aim was for kids to interact by playing together to try and solve the questions, having the system handheld would encourage team play.

Other Projects

Apart from the final assigment project above, there were 5 other labs where other arduino systems were built.

Have a look at the final build of each lab below where I reflect on the learning outcomes.

Lab 1 - Buttons and L.E.D's

Start from the very beginning. This first lab introduced us to the Arduino board, breadboards, Arduino components and the Arduino IDE where the code is generated and modified.

The final outcome of the lab was potentiometer controlling the brightness of an L.E.D. when a push button was pressed.

Overall I am happy with my output from this lab, considering I went into it completely un-educated in Arduino, I think this basic exercise was very beneficial as controlling a light with buttons and potentiometers is almost in every electronic device

Lab 2 - Potentiometers and RGB L.E.D's

The second lab concentrated on Power Wave Manipulation. This is the process of converting analog input from the potentiometer to digital output on the RGB L.E.D. using mapping to measure the voltage and convert it to the values fo the RGB L.E.D. A temperature sensor was also used to test the analog input.

Looking back I can see the logic in progressing through the labs. Each one builds on the previous work and adds new elements. The RGB LED is in interesting component. It can re-create most colours by combining the Red Green and Blue. I can see this component becoming part of my Go-to prototyping kit.

Although I enjoyed the lab, the programming of the code was very challenging. I got there, but I plan to re-visit this exercise and have another go.

Lab 5 - LCD displays and Code functions

Skipping forward a little. The final lab before the major assignment introduced us to LCD displays on the hardware side and Functions on the coding side.

The system in the image here is the prototype equivalent of a Magic 8 Ball. When the system is rocked, it generates a random response on the LCD screen. In the coding, functions are used to generate the random reply.

Even after completing the lab I am looking at the code and it surprises me that I was able to figure this all out. It seems complex but becasue we were introdcued to each of the concepts gradually over time, it was not as daunting to come up with the code.
